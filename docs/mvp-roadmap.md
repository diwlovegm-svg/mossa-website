# MVP Roadmap

## Phase 0: Pre-build

เป้าหมาย: ล็อกข้อมูลและเตรียม asset ก่อนทำเว็บ

งานที่ต้องทำ:

- ยืนยัน Google Maps URL
- รวบรวมรูปจริงชุดแรก
- ยืนยันสิทธิ์ Membership ทุกแพ็กเกจ
- ยืนยันข้อความเงื่อนไข Trial 3 วัน
- ยืนยันว่าข้อมูลบริษัทคู่สัญญาส่วนใดแสดง Public ได้

ผลลัพธ์:

- Content set เวอร์ชันแรก
- Image set เวอร์ชันแรก
- ข้อความเงื่อนไขที่ใช้ขึ้นเว็บได้

## Phase 1: Public Web App MVP

เป้าหมาย: ลูกค้าหาข้อมูลและติดต่อ MOSSA ได้เอง

แนวทาง Prototype:

- Header ใช้ `MOSSA Sport Society`
- SEO title ใช้ `MOSSA Sport Society Rayong`
- ข้อมูลใช้ไฟล์ JSON ก่อน ยังไม่ทำ Admin เต็ม
- ตารางคลาสเป็นรูปตารางรายเดือน เพื่อให้เปลี่ยนไฟล์ได้ง่าย
- บริษัทคู่สัญญาใช้ตัวอย่างจริง 5 บริษัท ได้แก่ PTT GC, AGC วีนิไทย, IRPC, Dow Chemical, SCG Chemicals
- แสดงที่อยู่ก่อน และซ่อนปุ่ม Google Maps จริงจนกว่าจะมีลิงก์ที่ยืนยันแล้ว
- ดีไซน์สปอร์ตเข้ม มืออาชีพ ใช้ม่วง / ส้ม / ดำ / ขาว

ฟีเจอร์:

- หน้าแรก
- บริการและราคา
- การ์ดบริการกดดูรายละเอียดพร้อมราคาในหน้าเดียว
- เวลาเปิดบริการ
- ตารางคลาส
- ทดลองฟรี 3 วัน
- บริษัทคู่สัญญาแบบค้นหา
- โปรโมชัน
- ติดต่อ
- FAQ
- Sticky CTA บนมือถือ

Acceptance:

- เปิดบนมือถือแล้วเห็น CTA หลักในจอแรก
- ราคาหลักทุกบริการแสดงครบ
- เวลาเปิดปิดแยกตามบริการ
- ปุ่ม LINE, Facebook, โทร ใช้งานได้
- ไม่มีข้อมูลภายในหลุดบนหน้า Public

## Phase 2: Admin Content Management

เป้าหมาย: แอดมินแก้ข้อมูลเองได้

ฟีเจอร์:

- แก้บริการ
- แก้ราคา
- แก้เวลาเปิดปิด
- แก้ตารางคลาส
- แก้โปรโมชัน
- แก้บริษัทคู่สัญญา
- แก้รูปภาพ
- เปิด/ปิดบริการหรือโปรโมชัน

Acceptance:

- แอดมินแก้ราคาแล้วหน้าเว็บอัปเดต
- แอดมินซ่อนบริการได้โดยไม่ลบข้อมูล
- แอดมินเพิ่มโปรโมชันได้
- แอดมินแก้ตารางคลาสรายเดือนได้

## Phase 3: Future Features

เลื่อนทำหลัง MVP ใช้งานจริง

- ระบบจองสนามออนไลน์
- ระบบเช็กคิวสนามว่าง
- ระบบจอง PT
- ระบบจองคลาส
- ระบบชำระเงินออนไลน์
- ระบบสมาชิก Login
- ระบบสะสมแต้ม
- ระบบออกใบเสร็จ/บิล

## Suggested Tech Direction

เวอร์ชันแรกควรเลือกเครื่องมือที่ทีมแก้ข้อมูลได้ง่าย

### Option A: Static Web App + Google Sheet + Apps Script

เหมาะสำหรับเริ่มเร็ว ค่าใช้จ่ายต่ำ และแอดมินแก้ข้อมูลผ่าน Sheet ได้

เหมาะกับ:

- บริการ
- ราคา
- เวลาเปิดปิด
- ตารางคลาส
- บริษัทคู่สัญญา
- ข้อมูลทดลองฟรี 3 วัน

ข้อควรระวัง:

- ต้องจัด permission Sheet ให้ดี
- ต้องแยกข้อมูล Public กับ Internal
- ต้องมี validation กันข้อมูลผิดรูปแบบ

### Option B: Web App + Database + Admin Panel

เหมาะเมื่อระบบใหญ่ขึ้นและมีหลายคนใช้งานหลังบ้าน

เหมาะกับ:

- Booking
- Payment
- Member login
- Audit log

ข้อเสนอ:

เริ่มด้วย Option A ใน MVP แล้วออกแบบ data structure ให้ย้ายไป Option B ได้ในอนาคต

## Development Checklist

- สร้าง project ใหม่
- วาง design system สี ฟอนต์ ปุ่ม และ CTA
- สร้าง seed data จาก `data-structure.md`
- สร้างหน้า Public
- เชื่อม data source
- ทำระบบค้นหาบริษัทคู่สัญญา
- ทำหลังบ้านแก้ข้อมูล

- เตรียม deploy

## Definition of Done

MVP ถือว่าเสร็จเมื่อ:

- ลูกค้าดูข้อมูลบริการ ราคา เวลา ตารางคลาส และโปรโมชันได้จากมือถือ
- ลูกค้าเห็นเงื่อนไขทดลองฟรี 3 วันชัดเจน
- ลูกค้าเข้าใจว่าต้องนำบัตรประชาชนมายื่นที่เคาน์เตอร์
- ลูกค้าค้นหาบริษัทคู่สัญญาได้
- ลูกค้ากด LINE / โทร / Inbox ได้
- แอดมินแก้ข้อมูลหลักได้
- ข้อมูลที่ยังรอยืนยันถูกแสดงแบบไม่ยืนยันเกินจริง
- ไม่มีข้อมูลภายในบนหน้า Public
